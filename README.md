# RPA 流程机器人

一款基于 Python + Tkinter 开发的 **Windows 桌面 RPA（机器人流程自动化）工具**，通过图像识别（OpenCV 模板匹配）实现 GUI 自动化操作，无需目标程序的 API 或源代码即可完成流程编排与批量执行。

> **版本**: v7.39 | 技术栈: Python 3.8+ / OpenCV / Tkinter / WinAPI | 许可证: MIT

---

## 功能概述

本工具采用**可视化流程编辑器**设计，通过拖拽和参数配置即可创建自动化流程，无需编写代码。支持**前后台双模式**——前台模式直接操作屏幕，后台模式通过 Windows API 向窗口发送消息，不抢占前台焦点。

### 节点类型一览（16种）

| 节点类型 | 功能说明 |
|---------|---------|
| **点击** | 基于图像模板匹配的自动点击，支持左键/右键/双击/仅识别，支持前后台 |
| **等待** | 等待图片出现 / 窗口出现 / 窗口消失，支持超时重试或跳过 |
| **输入** | 模拟键盘输入文本，支持**直接输入**和**数据变量**两种来源 |
| **按键** | 模拟按键操作：F1-F12、Enter、Tab 等特殊键 + Ctrl/Alt/Shift 组合键 |
| **滚轮** | 模拟鼠标滚轮滚动 |
| **延时** | 固定时长等待 |
| **条件分支** | 基于图像是否存在（或变量条件）进行分支判断，执行不同子流程 |
| **普通循环** | 按次数循环 或 基于图像/变量条件的循环，支持退出条件 |
| **数据循环** | 读取 CSV/Excel 文件，逐行循环执行子流程，支持退出条件 |
| **标签** | 流程中的标记点，配合跳转节点实现流程跳转 |
| **跳转** | 无条件跳转到指定标签位置，支持跨子流程跳转 |
| **窗口** | 窗口操作：激活前台/设为后台目标/最大化/最小化/置顶/取消置顶 |
| **文件** | 打开文件或程序 |
| **OCR** | 调用 Umi-OCR 服务进行文字识别，支持识别后自动点击指定区域 |
| **暂停** | 流程暂停，弹出确认对话框等待用户操作 |
| **变量管理** | 在流程中定义变量（字符串/数字/布尔），供后续节点使用 |
| **变量计算** | 对变量进行四则运算和常用函数计算，结果保存到指定变量 |
| **退出** | 立即终止整个流程（常用于条件分支的退出路径） |

## 核心特性

### 视觉引擎

- **图像模板匹配**: 基于 OpenCV `matchTemplate`（TM_CCOEFF_NORMED 算法）
- **双通道匹配模式**:
  - **普通模式**: 仅匹配形状灰度图，速度快
  - **颜色敏感模式**: 同时匹配颜色（BGR）和形状（灰度），通过 LAB Delta E 平均值校验剔除颜色不符的伪匹配，精度更高
- **多目标识别**: 支持查找第 N 个匹配项
- **重叠结果过滤**: 基于模板尺寸的比例过滤相邻重复结果
- **颜色校验辅助**: 提供 `#RRGGBB` 和 `R,G,B` 两种格式的颜色解析，支持在 PIL 和 OpenCV 图像中向量化检测指定颜色区域
- **前后台双模式**:
  - **前台模式**（`pyautogui.screenshot`）: 截取全屏，适用于无需后台的场景
  - **后台模式**（`PrintWindow + BitBlt`）: 截取指定窗口客户区，窗口最小化也可操作

### 执行引擎

- **独立线程运行**: 流程在后台线程中执行，不阻塞 UI
- **多级停止机制**: `StopException` + `threading.Event`，支持 UI 停止按钮和 Ctrl+Q 全局热键
- **超时重试/跳过**: 点击或等待超时时弹窗让用户选择重试或跳过
- **数据变量系统**: 运行时变量表，支持流程中动态传递值
- **变量管理节点**: 支持字符串、数字、布尔三种类型的变量定义
- **变量计算节点**: 四则运算 + 常见函数（abs/round/min/max/len/sum/pow），结果保存到指定变量
- **循环退出条件**: 普通循环和数据循环均支持「图片条件」和「变量条件」两种退出条件
- **子流程支持**: 条件分支、循环、数据循环内部支持嵌套子流程

### 窗口驱动层

- **后台操作**: 通过 Windows API（`PostMessage`）向窗口发送鼠标、键盘消息，**不抢占前台焦点**
- **后台截图**: 依次尝试 `PrintWindow(PW_RENDERFULLCONTENT)` → `PrintWindow(PW_CLIENTONLY)` → `BitBlt`，自动降级保证最大兼容性
- **GDI 资源管理**: 使用上下文管理器自动释放 HDC、Bitmap 等资源，防止内存泄漏
- **窗口容错**: 记录后台窗口标题，窗口丢失后自动重试找窗
- **坐标转换**: 支持屏幕坐标与窗口客户区坐标的相互转换
- **窗口等待**: 等待窗口出现或消失（基于 `EnumWindows` 轮询，不区分大小写模糊匹配）

### 多选复制粘贴（v7.39 新增）

- 支持 `Ctrl+左键` 和 `Shift+左键` 多选流程步骤
- `Ctrl+C` 复制选中步骤（或右键菜单 → 复制）
- `Ctrl+V` 粘贴插入到选中步骤之后（或右键菜单 → 粘贴插入）
- 粘贴时自动重新生成节点 ID，不与其他步骤冲突
- 支持跨子流程（条件分支/循环内部）的复制粘贴

### 数据循环

- **支持格式**: CSV、Excel（通过 pandas + openpyxl）
- **循环模式**: 向下取数（从头到尾）、向上取数（从尾到头）
- **变量注入**: 通过 `数据名称.字段名` 格式在输入节点和 OCR 节点中动态引用数据
- **退出条件**: 支持图片条件和变量条件两种退出机制

### OCR 集成

- 集成 **Umi-OCR** HTTP API 接口
- 支持全文识别、区域识别
- 支持识别结果中的关键字匹配后自动点击
- 关键字支持从数据变量获取

### 热键与快捷键

| 按键 | 功能 |
|------|------|
| `Ctrl+Q` | 全局停止流程（可配置） |
| `Ctrl+C` | 复制选中的流程步骤（流程编辑器中） |
| `Ctrl+V` | 粘贴插入流程步骤 |

**创建流程**:
1. 点击 **「新建」** 创建新流程
2. 在右侧「步骤参数」面板的 **节点类型** 下拉框中选择节点类型
3. 在下方的参数区域配置节点参数
4. 点击 **「插入步骤」** 按钮添加到左侧的流程步骤列表
5. 在左侧列表中拖拽调整步骤顺序

**运行流程**: 点击 **「开始运行」** → 程序自动最小化 → 按 `Ctrl+Q` 随时停止

## 使用示例

### 示例 1：简单的图片点击流程

```
1. [窗口] 激活目标窗口
2. [点击] 点击"开始"按钮图片
3. [等待] 等待"完成"图片出现
4. [点击] 点击"确认"按钮
```

### 示例 2：数据循环批量处理

```
1. [窗口] 激活目标窗口
2. [数据循环] 读取 data.csv
   - 数据名称: 客户
   - 循环模式: 向下取数
   - 循环体:
     a. [输入] 数据变量: 客户.姓名
     b. [输入] 数据变量: 客户.电话
     c. [点击] 点击"提交"按钮
     d. [延时] 等待 1 秒
3. [暂停] 流程完成
```

### 示例 3：带变量管理的条件流程

```
1. [变量管理] 定义变量：计数 = 0（数字型）
2. [条件分支] 判断"开始任务"图片是否存在
   - 是: [点击] 点击"开始" → 进入循环
   - 否: [退出] 直接结束流程
3. [普通循环] 按次数循环 5 次
   - 循环体:
     a. [输入] 输入文本
     b. [点击] 点击"提交"
     c. [变量计算] 计数 = 计数 + 1
4. [暂停] 流程完成
```

## 安装与使用

### 系统要求

- Windows 10/11（依赖 Win32 API，不支持 Linux/macOS）
- Python 3.8+
- 如需 OCR 功能：安装并启动 [Umi-OCR](https://github.com/hiroi-sora/Umi-OCR)（默认监听 `http://127.0.0.1:1224`）

### 安装依赖

```bash
pip install -r requirements.txt
```

### 启动程序

```bash
python main.py
```

### 保存/加载流程

- **「保存 JSON」** — 将流程配置保存为 JSON 文件
- **「加载 JSON」** — 加载已有的流程配置

### 前后台双模式

| 模式 | 截图方式 | 操作方式 | 适用场景 |
|------|---------|---------|---------|
| **前台** | `pyautogui.screenshot` | `pyautogui.click/type/scroll` | 无需后台隐藏，UI 可见 |
| **后台** | `PrintWindow → BitBlt` | `PostMessage` 发送 WM_* 消息 | 窗口最小化、不干扰用户 |

## 常见问题

### 1. 图像识别匹配不到目标

- 降低「相似度阈值」（默认 0.95，可调至 0.7-0.8）
- 重新截图模板（确保与目标区域一致）
- 检查目标窗口是否被遮挡（后台模式不受影响）

### 2. 后台模式不生效

- 确认已通过「窗口」节点的「激活(设为后台目标)」设置目标窗口
- 某些应用程序不支持 `PrintWindow` 截图（大多数 Win32 原生应用正常，部分 DirectX/WebView 需前台模式）

### 3. 打包后 exe 被杀毒软件误报

- PyInstaller 打包的程序可能被误报为病毒，可添加信任或购买代码签名证书

### 4. OCR 功能不工作

- 确保已安装并启动 [Umi-OCR](https://github.com/hiroi-sora/Umi-OCR)
- 确认默认端口 1224 未被占用
