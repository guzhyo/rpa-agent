# RPA Agent（AI 版）

一款基于 Python + Tkinter 开发的 **Windows 桌面 RPA（机器人流程自动化）工具**，在传统 RPA 的图像识别、流程编排能力基础上，新增 **AI 决策节点**，让 AI 接管流程中的智能判断环节。

> **版本**: v7.39 | 技术栈: Python 3.8+ / OpenCV / Tkinter / WinAPI / AI API | 许可证: MIT

---

## 功能概述

采用**可视化流程编辑器**设计，通过拖拽和参数配置即可创建自动化流程。支持**前后台双模式**——前台模式直接操作屏幕，后台模式通过 Windows API 向窗口发送消息，不抢占前台焦点。

### 节点类型一览（19种）

| 节点类型 | 功能说明 |
|---------|---------|
| **AI 决策** | 调用 AI（支持本地 llamafile / OpenAI / Claude 等）分析截图或文本，结果保存到变量 |
| **点击** | 基于图像模板匹配的自动点击，支持左键/右键/双击/仅识别，支持前后台 |
| **等待** | 等待图片出现 / 窗口出现 / 窗口消失，支持超时重试或跳过 |
| **输入** | 模拟键盘输入文本，支持**直接输入**和**数据变量**两种来源 |
| **按键** | 模拟按键操作：F1-F12、Enter、Tab 等特殊键 + Ctrl/Alt/Shift 组合键 |
| **滚轮** | 模拟鼠标滚轮滚动 |
| **延时** | 固定时长等待 |
| **条件分支** | 基于图像是否存在（或变量条件）进行分支判断，执行不同子流程 |
| **普通循环** | 按次数循环 或 基于图像/变量条件的循环，支持退出条件 |
| **数据循环** | 读取 CSV/Excel 文件，逐行循环执行子流程，支持退出条件 |
| **标签** | 流程中的标记点，配合跳转节点实现流程跳转 |
| **跳转** | 无条件跳转到指定标签位置，支持跨子流程跳转 |
| **窗口** | 窗口操作：激活前台/设为后台目标/最大化/最小化/置顶/取消置顶 |
| **文件** | 打开文件或程序 |
| **OCR** | 调用 Umi-OCR 服务进行文字识别，支持识别后自动点击指定区域 |
| **暂停** | 流程暂停，弹出确认对话框等待用户操作 |
| **变量管理** | 在流程中定义变量（字符串/数字/布尔），供后续节点使用 |
| **变量计算** | 对变量进行四则运算和常用函数计算，结果保存到指定变量 |
| **退出** | 立即终止整个流程（常用于条件分支的退出路径） |

## AI 决策节点

本项目的核心特性，让 RPA 流程具备 AI 智能判断能力。

### 两种输入模式

| 模式 | 说明 | 典型场景 |
|------|------|---------|
| **图片识别** | 截图当前屏幕或指定区域，发送给多模态 AI 分析 | 识别验证码、判断页面状态、分析图表内容 |
| **文本语义** | 将流程运行时的变量、OCR 结果等文本发送给 LLM 分析 | 日志分析、语义判断、内容提取 |

### 支持的 API

兼容 OpenAI 格式的 API 均可使用：

- **llamafile**（本地运行，无需联网，无需 API Key）
- **OpenAI GPT-4o**（云端多模态）
- **Anthropic Claude Sonnet**（云端多模态）
- **任何兼容 OpenAI API 的本地或云端服务**

### 配置参数

| 参数 | 说明 |
|------|------|
| **输入模式** | 图片识别 / 文本语义 |
| **API 地址** | 兼容 OpenAI 格式，默认 http://localhost:8080/v1/chat/completions |
| **模型名称** | 如 gpt-4o，llamafile 可留空 |
| **API Key** | 本地模型可空 |
| **提示词** | 告诉 AI 要做什么判断（多行文本框） |
| **文本来源** | 文本语义模式下，输入变量名或直接输入文本 |
| **截图区域** | 图片识别模式下，指定截图区域，留空为全屏 |
| **输出变量** | AI 返回的结果保存到指定变量名 |
| **超时时间** | API 调用超时（秒） |

### 流程示例

```
[窗口] 激活目标页面
[点击] 点击"查询"按钮
[等待] 等待结果加载
[AI 决策] 图片识别：判断页面是否有数据
  → 结果保存到 has_data 变量
[条件分支] 变量条件：has_data 包含"有数据"
  ├─ 是 → [数据循环] 处理数据
  └─ 否 → [暂停] 提示无数据
```

```
[AI 决策] 文本语义：分析 OCR 结果中是否包含异常信息
  → 结果保存到 analysis 变量
[条件分支] 变量条件：analysis 包含"异常"
  ├─ 是 → [暂停] 人工处理
  └─ 否 → [延时] 继续下一个
```

## 核心特性

### 视觉引擎

- **图像模板匹配**: 基于 OpenCV `matchTemplate`（TM_CCOEFF_NORMED 算法）
- **双通道匹配模式**: 普通模式（灰度）/ 颜色敏感模式（BGR + LAB Delta E 校验）
- **多目标识别**: 支持查找第 N 个匹配项
- **重叠结果过滤**: 基于模板尺寸的比例过滤相邻重复结果
- **前后台双模式**: 前台（`pyautogui.screenshot`）/ 后台（`PrintWindow + BitBlt`）

### 执行引擎

- **独立线程运行**: 流程在后台线程中执行，不阻塞 UI
- **多级停止机制**: `StopException` + `threading.Event`，支持 UI 停止按钮和 Ctrl+Q 全局热键
- **超时重试/跳过**: 点击或等待超时时弹窗让用户选择重试或跳过
- **数据变量系统**: 运行时变量表，支持流程中动态传递值
- **后台窗口容错**: 窗口丢失时自动重试恢复，恢复失败降级为前台模式

### 窗口驱动层

- **后台操作**: 通过 Windows API（`PostMessage`）向窗口发送鼠标、键盘消息，**不抢占前台焦点**
- **后台截图**: 依次尝试 `PrintWindow(PW_RENDERFULLCONTENT)` → `PrintWindow(PW_CLIENTONLY)` → `BitBlt`
- **窗口容错**: 记录后台窗口标题，窗口丢失后自动重试找窗

### 剪切/复制/粘贴/撤消

- `Ctrl+X` 剪切、`Ctrl+C` 复制、`Ctrl+V` 粘贴插入、`Ctrl+Z` 撤消
- 支持多选、跨子流程复制粘贴
- 右键菜单集成全部操作

## 安装与使用

### 系统要求

- Windows 10/11（依赖 Win32 API）
- Python 3.8+
- 如需 OCR 功能：安装并启动 [Umi-OCR](https://github.com/hiroi-sora/Umi-OCR)
- 如需 AI 决策：启动 llamafile 或其他兼容 OpenAI API 的服务

### 安装依赖

```bash
pip install -r requirements.txt
```

### 启动程序

```bash
python main.py
```

## 常见问题

### 1. AI 决策节点连接失败

- 确认 API 服务已启动（如 llamafile 是否运行）
- 检查 API 地址和端口是否正确
- 如果使用云端 API，确认网络连通和 API Key 有效

### 2. 图像识别匹配不到目标

- 降低相似度阈值（默认 0.95，可调至 0.7-0.8）
- 重新截图模板

### 3. 后台模式不生效

- 确认已通过「窗口」节点设置后台目标
- 某些应用不支持 `PrintWindow`，可切换到前台模式
